# Evidec — Evidence Engine for Decision Making

> 意思決定のための証拠生成フレームワーク（Evidence Generation Framework for Decision Making）

## 1. プロジェクト概要（What is Evidec?）

**Evidec** は、データ分析を **"計算で終わらせず、意思決定につなぐ"** ための Python OSS です。

A/B テストや統計検定、サンプルサイズ算出といった分析手法は既に多く存在しています。しかし実務では、

```
実験設計 → 検定 → 結果解釈 → 意思決定（Go / No-Go）→ 根拠提示（Evidence Report）
```

が **バラバラなツール・スクリプトで管理されている** ことが多く、組織的な意思決定の質を保つのが難しいという問題があります。

**Evidec** はこの断絶を埋め、**実験から意思決定までの流れを一貫して扱えるフレームワーク**を提供します。

## 2. 解決したい課題（Problem）

### ● 分析はできるが、意思決定につながらない

多くの OSS は「検定」や「統計計算」までしか提供しません。最終的に必要なのは **意思決定の根拠** です。

### ● A/B テストごとに毎回スクリプトをスクラッチで作る非効率

実験設計・集計・検定・判定が毎回バラバラに実装されているため再現性が低い。

### ● 判断ルールが個人依存で、組織的に標準化されていない

**Evidec** は意思決定のロジックを **明文化し、コードとして共有可能** にします。

## 3. Evidec が提供する価値（Value Proposition）

### 1. 実験設計 → 証拠生成 → 意思決定までを一貫提供

分断されていた分析フローを統合し、判断可能な証拠を自動生成します。

### 2. 意思決定の再現性と品質を向上

判断ロジックを **DecisionRule** としてコード化し、チーム全体で統一した基準を使えます。

### 3. 分析者・PdM・経営者が同じ言語で会話できる基盤

**Evidec** が生成する **Evidence Report** は、非エンジニアでも理解可能な形式を目指します。

## 4. 主な機能スコープ（Scope）

### Design（実験設計）
- サンプルサイズ計算（Power, MDE）
- 実験期間の推計
- メトリクス定義

### Validation（統計検証）
- A/B テスト（t検定・比率検定・ベイズ手法など）
- 信頼区間計算
- 多重検定補正

### Decision（意思決定ロジック）
- **DecisionRule** による Go / No-Go / Inconclusive 判定
- 判定理由の自動生成
- 経営判断で使える基準作りの支援

### Reporting（証拠化）
- **Evidence Report** の生成
- 統計結果の要約
- 意思決定に必要な要点を簡潔にまとめるドキュメント出力

## 5. 名前の由来（Naming Identity）

**名称：Evidec（エビデック）**

### 由来：
**Evidence（証拠） × Decision（意思決定）**

### 特徴：
- コンセプトを名前だけで説明できる
- テック感のある語感（Codec / Logic / Tech）
- 会話で呼びやすい「エビデック」
- 検索性・ブランド性が高い
- コードでの利用イメージも自然：`import evidec as ed`

## 6. タグライン（Tagline）

> **"Turn Data into Decision."**
> **データを、意思決定の「確かな証拠」へ。**

## 7. 初期 API イメージ（Interface Image）

```python
from evidec import Experiment

exp = Experiment(
    name="cta_test",
    metric="ctr",
    effect="increase",
    alpha=0.05,
    min_effect=0.02,
)

exp.fit(a_data, b_data)
result = exp.decision()

print(result.summary())
```

**設計思想：** 分析ではなく、意思決定へ一直線に向かうコード。
